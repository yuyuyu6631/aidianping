# aidianping v2

`aidianping v2` 是一个全新仓库，用来重建 AI 工具目录产品，而不是继续在旧仓库上做高风险重构。

当前阶段只完成基础工作：

- 建立新仓库的 monorepo 结构
- 迁移可复用的领域模型和接口契约思路
- 输出迁移说明和下一步开发基线文档
- 明确哪些旧资产有价值，哪些不能直接复用

## 目录结构

```text
apps/
  api/
  web/
packages/
  contracts/
docs/
```

## 当前已落地的基础资产

- `packages/contracts`
  - 共享领域模型
  - 列表页、详情页、AI 搜索、评论评分相关接口契约
- `docs/00-可复用资产迁移说明.md`
  - 说明从旧项目迁移了什么，为什么有价值
- `docs/01-v2开发基础基线.md`
  - 说明 v2 仓库的开发边界、角色分工和启动顺序

## 不直接复用的旧资产

- 旧首页、旧列表页、旧详情页的具体实现
- 旧样式文件和页面布局代码
- 旧设计稿中的乱码和失真文案
- 为兼容旧结构而存在的过渡逻辑

## 下一步建议

1. 在 `apps/web` 初始化 Next.js 应用骨架
2. 在 `apps/api` 初始化后端服务骨架
3. 先接 `packages/contracts` 里的共享契约，再补 mock 数据
4. 先做首页、列表页、详情页三张静态高保真骨架，再接 API
