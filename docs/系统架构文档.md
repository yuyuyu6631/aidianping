# 星点评 V1 系统架构文档

## 1. 项目概述

### 1.1 项目名称
星点评（AI 工具集合平台）

### 1.2 项目背景
聚合 AI 工具，支持搜索、筛选、收藏、评分、评论和个人管理，帮助用户快速发现和管理 AI 工具；同时通过内部后台支持工具数据的持续维护。

### 1.3 核心功能
| 模块 | 功能 |
|------|------|
| 首页 | 极简风格首页、精选工具、专题推荐模块、搜索入口 |
| 工具列表/更多页 | 搜索、分类筛选、标签筛选、排序、工具卡片展示、专题更多页承接 |
| 工具详情页 | 工具信息、同类推荐、评分总览、评论列表、提交评分评论 |
| 个人中心 | 用户信息、收藏工具、评论记录、退出登录、注销账号 |
| 内部管理后台 | AI 工具 CRUD、分类标签维护、专题配置、上下架、轻量审核 |

### 1.4 用户角色
| 角色 | 权限 |
|------|------|
| 游客 | 浏览工具、搜索、查看详情和评论 |
| 注册用户 | 收藏工具、提交评分评论、管理个人收藏和评论 |
| 内部运营/管理员 | 维护工具、分类、标签、专题与发布状态 |

### 3.2 Prisma Schema

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}

model User {
  id            Int            @id @default(autoincrement())
  email         String         @unique
  password      String
  nickname      String         @default("")
  avatar        String         @default("")
  role          String         @default("user") // user | admin
  createdAt     DateTime       @default(now())
  updatedAt     DateTime       @updatedAt

  favorites     Favorite[]
  reviews       Review[]
  refreshTokens RefreshToken[]

  @@map("users")
}

model Tool {
  id            Int            @id @default(autoincrement())
  name          String         @unique
  logo          String         @default("")
  shortDesc     String         @db.Text
  fullDesc      String         @db.LongText
  primaryUsage  String         @default("")
  officialUrl   String         @default("")
  popularity    Int            @default(0)
  publishTime   DateTime       @default(now())
  status        String         @default("draft") // draft | published | archived
  isFeatured    Boolean        @default(false)
  createdAt     DateTime       @default(now())
  updatedAt     DateTime       @updatedAt

  categories    ToolCategory[]
  tags          ToolTag[]
  topics        ToolTopic[]
  reviews       Review[]
  favorites     Favorite[]

  @@index([status])
  @@index([isFeatured])
  @@map("tools")
}

model Category {
  id          Int            @id @default(autoincrement())
  name        String         @unique
  slug        String         @unique
  description String         @default("")
  createdAt   DateTime       @default(now())

  tools       ToolCategory[]

  @@map("categories")
}

model ToolCategory {
  toolId     Int
  categoryId Int
  tool       Tool     @relation(fields: [toolId], references: [id], onDelete: Cascade)
  category   Category @relation(fields: [categoryId], references: [id], onDelete: Cascade)

  @@id([toolId, categoryId])
  @@map("tool_categories")
}

model Tag {
  id    Int       @id @default(autoincrement())
  name  String    @unique
  slug  String    @unique

  tools ToolTag[]

  @@map("tags")
}

model ToolTag {
  toolId Int
  tagId  Int
  tool   Tool @relation(fields: [toolId], references: [id], onDelete: Cascade)
  tag    Tag  @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([toolId, tagId])
  @@map("tool_tags")
}

model Topic {
  id          Int         @id @default(autoincrement())
  name        String      @unique
  slug        String      @unique
  description String      @default("")
  sortOrder   Int         @default(0)
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt

  tools       ToolTopic[]

  @@map("topics")
}

model ToolTopic {
  toolId  Int
  topicId Int
  tool    Tool  @relation(fields: [toolId], references: [id], onDelete: Cascade)
  topic   Topic @relation(fields: [topicId], references: [id], onDelete: Cascade)

  @@id([toolId, topicId])
  @@map("tool_topics")
}

model Favorite {
  id        Int      @id @default(autoincrement())
  userId    Int
  toolId    Int
  createdAt DateTime @default(now())

  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  tool      Tool     @relation(fields: [toolId], references: [id], onDelete: Cascade)

  @@unique([userId, toolId])
  @@map("favorites")
}

model Review {
  id        Int      @id @default(autoincrement())
  userId    Int
  toolId    Int
  content   String   @db.Text
  rating    Int?
  status    String   @default("visible") // visible | hidden
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  tool      Tool     @relation(fields: [toolId], references: [id], onDelete: Cascade)

  @@unique([userId, toolId])
  @@map("reviews")
}

model RefreshToken {
  id        Int       @id @default(autoincrement())
  userId    Int
  tokenHash String    @unique
  expiresAt DateTime
  revokedAt DateTime?
  createdAt DateTime  @default(now())

  user      User      @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@index([expiresAt])
  @@map("refresh_tokens")
}
```

## 5. 模块设计

### 5.1 Auth 模块

**职责**：用户认证与授权

**核心功能**：
- 注册（邮箱 + 密码）
- 登录（邮箱 + 密码）
- Token 刷新
- 登出

**安全设计**：
- 密码使用 bcrypt 加密
- Access Token 有效期 15 分钟
- Refresh Token 有效期 7 天，哈希后存入数据库
- Refresh Token 支持撤销
- 后台接口通过 `AdminGuard` 与普通用户接口隔离

### 5.2 User 模块

**职责**：用户信息管理

**核心功能**：
- 获取当前用户信息
- 更新用户信息（昵称、头像）
- 注销账号（级联删除收藏和评论）

### 5.3 Tool 模块

**职责**：AI 工具管理

**核心功能**：
- 首页精选工具查询
- 工具列表查询（搜索 + 筛选 + 排序 + 分页）
- 专题更多页查询
- 工具详情查询
- 同类推荐

### 5.4 Category 模块

**职责**：工具分类管理

**核心功能**：
- 全量分类列表查询
- 后台分类 CRUD

### 5.5 Tag 模块

**职责**：工具标签管理

**核心功能**：
- 全量标签列表查询
- 后台标签 CRUD

### 5.6 Topic 模块

**职责**：首页专题与更多页管理

**核心功能**：
- 首页专题推荐列表
- 专题更多页工具查询
- 专题排序与展示控制

### 5.7 Favorite 模块

**职责**：用户收藏管理

**核心功能**：
- 用户收藏列表
- 添加收藏（幂等）
- 取消收藏（幂等）

### 5.8 Review 模块

**职责**：评论管理

**核心功能**：
- 工具评论列表
- 提交评分评论
- 用户评论列表

**业务规则**：
- 评论内容必填，最多 2000 字符
- 评分为 1-5 整数，可选
- 一个用户对同一工具只允许一条评论，重复提交视为更新

### 5.9 Admin 模块

**职责**：内部运营后台管理

**核心功能**：
- 工具新增、编辑、删除、上下架
- 分类、标签、专题 CRUD
- 首页精选配置
- 评论查看与隐藏

**审核策略**：
- V1 仅保留轻量状态位
- 不做复杂审批流
- 通过 `draft`、`published`、`archived` 控制可见性

## 6. 认证与授权

### 6.1 JWT Token 结构

**Access Token Payload**：
```json
{
  "sub": 1,
  "email": "user@example.com",
  "role": "user",
  "type": "access",
  "iat": 1699999999,
  "exp": 1700000899
}
```

**Refresh Token Payload**：
```json
{
  "sub": 1,
  "type": "refresh",
  "iat": 1699999999,
  "exp": 1700604799
}
```

### 6.2 守卫策略

| 守卫 | 用途 |
|------|------|
| JwtAuthGuard | 保护需要登录的接口 |
| PublicGuard | 跳过认证检查 |
| JwtRefreshGuard | 仅验证 Refresh Token |
| AdminGuard | 保护后台管理接口 |

## 7. 错误处理

### 7.1 错误码规范

| 错误码 | 说明 |
|--------|------|
| 400 | 参数错误 / 参数校验失败 |
| 401 | 未认证 / Token 无效或过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 500 | 服务器内部错误 |

### 7.2 错误响应格式

```json
{
  "code": 400,
  "message": "参数校验失败",
  "errors": [
    { "field": "email", "message": "邮箱格式不正确" }
  ]
}
```

### 10.1 环境要求
- Node.js 18+
- MySQL 8.0+
- Redis（可选）
- Nginx

### 10.2 进程管理
- 生产环境使用 PM2 或 Docker 部署
- 支持多实例横向扩展
- 后台与前台可共享后端服务，通过路由和权限隔离

### 10.3 环境配置
| 环境 | 用途 |
|------|------|
| development | 本地开发 |
| staging | 预发布测试 |
| production | 生产环境 |
