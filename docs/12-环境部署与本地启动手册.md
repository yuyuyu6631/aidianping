## 2. 环境要求

### 2.1 基础环境

- Node.js 18+
- npm / pnpm
- MySQL 8.0+
- Redis（可选）
- Git

### 2.2 推荐目录

```text
d:/workspace/codespace/
├── aidianping/
├── aidianping-fe/
└── aidianping-be/
```

---

## 3. 本地环境变量

### 3.1 前端 `.env.local`

```bash
NEXT_PUBLIC_API_BASE_URL=http://localhost:3000/api
```

### 3.2 后端 `.env`

```bash
NODE_ENV=development
PORT=3000
DATABASE_URL=mysql://root:password@localhost:3306/aidianping
REDIS_URL=redis://localhost:6379
JWT_ACCESS_SECRET=your-access-secret-key
JWT_REFRESH_SECRET=your-refresh-secret-key
JWT_ACCESS_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d
SWAGGER_TITLE=星点评 API 文档
SWAGGER_DESCRIPTION=AI 工具集合平台 RESTful API
SWAGGER_VERSION=1.0.0
```

## 4. 数据库初始化

### 4.1 创建数据库

```sql
CREATE DATABASE aidianping DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 4.2 执行 migration

```bash
npx prisma migrate dev
```

### 4.3 导入种子数据

```bash
npx prisma db seed
```

---

## 5. 本地启动顺序

### 5.1 启动后端

```bash
cd d:/workspace/codespace/aidianping-be
npm install
npx prisma migrate dev
npx prisma db seed
npm run start:dev
```

### 5.2 启动前端

```bash
cd d:/workspace/codespace/aidianping-fe
npm install
npm run dev
```

### 5.3 验证地址

- 前端：http://localhost:3001 或项目实际端口
- 后端：http://localhost:3000
- Swagger：http://localhost:3000/api-docs

---

## 6. 联调环境建议

### 6.1 基本原则

- 前端联调环境固定指向测试后端
- 后端测试环境固定指向测试数据库
- 不允许前端直接连生产后端

### 6.2 环境分类

| 环境 | 用途 |
|------|------|
| local | 本地开发 |
| dev | 共享开发环境 |
| staging | 提测环境 |
| production | 生产环境 |

---

## 7. Redis 使用建议

V1 可选启用 Redis，用于：

- 分类缓存
- 标签缓存
- 专题页缓存

如果本地不启用 Redis：

- 后端必须允许降级运行
- 不应影响基础 CRUD 和联调

## 9. 常见问题排查

### 9.1 后端启动失败

先检查：

- `.env` 是否存在
- MySQL 是否启动
- `DATABASE_URL` 是否正确
- migration 是否执行

### 9.2 前端请求不到接口

先检查：

- `NEXT_PUBLIC_API_BASE_URL` 是否正确
- 后端端口是否启动
- 浏览器控制台是否有跨域错误

### 9.3 Swagger 打不开

先检查：

- 后端是否启动成功
- Swagger 路由是否注册
- 端口是否冲突

## 10. 部署最小方案

### 10.1 前端

- Next.js 独立部署
- Nginx 反向代理

### 10.2 后端

- NestJS 使用 PM2 或 Docker 启动

### 10.3 数据层

- MySQL 单实例
- Redis 可选

V1 不要求复杂云原生架构，先保证稳定可部署。
