# 13-数据字典与业务枚举表

## 1. 文档目标

统一项目内常用业务枚举、状态值和关键数据定义，避免前后端各自定义。

---

## 2. 用户相关

### 2.1 用户角色 `user.role`

| 值 | 含义 | 说明 |
|----|------|------|
| user | 普通用户 | 可使用前台功能 |
| admin | 管理员 | 可进入后台 |

---

## 3. 工具相关

### 3.1 工具状态 `tool.status`

| 值 | 含义 | 前台可见 | 后台可见 |
|----|------|----------|----------|
| draft | 草稿 | 否 | 是 |
| published | 已发布 | 是 | 是 |
| archived | 已归档 | 否 | 是 |

### 3.2 首页精选 `tool.isFeatured`

| 值 | 含义 |
|----|------|
| true | 首页精选工具 |
| false | 非首页精选工具 |

### 3.3 排序值 `sort`

| 值 | 含义 |
|----|------|
| popularity | 按热度排序 |
| newest | 按发布时间排序 |
| rating | 按评分排序，仅评论列表或预留场景可用，不属于当前 V1 工具列表 / 专题更多页筛选值 |

---

## 4. 评论相关

### 4.1 评论状态 `review.status`

| 值 | 含义 | 前台可见 |
|----|------|----------|
| visible | 可见 | 是 |
| hidden | 已隐藏 | 否 |

### 4.2 评论评分 `review.rating`

| 值 | 含义 |
|----|------|
| 1 | 很差 |
| 2 | 较差 |
| 3 | 一般 |
| 4 | 较好 |
| 5 | 很好 |

---

## 5. 分类与标签

### 5.1 分类 slug 规范

格式：

- 小写英文
- 单词之间使用 `-`

示例：

- `ai-writing`
- `ai-video`
- `ai-coding`

### 5.2 标签 slug 规范

示例：

- `free`
- `paid`
- `domestic`
- `overseas`
- `beginner`
- `efficiency`

---

## 6. 专题相关

### 6.1 专题 slug 规范

格式：

- 小写英文
- 推荐带业务后缀

示例：

- `ai-writing-tools`
- `ai-ppt-tools`
- `ai-video-tools`

### 6.2 专题排序 `topic.sortOrder`

| 值 | 含义 |
|----|------|
| 数字越小 | 排序越靠前 |

---

## 7. 分页默认值

| 字段 | 默认值 | 最大值 |
|------|--------|--------|
| page | 1 | - |
| pageSize | 20 | 100 |

---

## 8. 时间字段规范

统一使用 ISO 8601 字符串：

```text
2026-04-16T10:00:00Z
```

字段命名统一：

- `createdAt`
- `updatedAt`
- `publishTime`
- `expiresAt`

---

## 9. 布尔字段规范

统一使用 `true/false`，不使用：

- `0/1`
- `"yes"/"no"`
- `"Y"/"N"`

典型字段：

- `isFavorited`
- `isFeatured`

---

## 10. 空值规范

| 场景 | 规则 |
|------|------|
| 文本字段无值 | 返回空字符串 `""` 或 `null`，需在接口字典中固定 |
| 数组字段无值 | 返回空数组 `[]` |
| 对象字段无值 | 返回 `null` |
| 分页列表无数据 | `list: []`，不是 `null` |
