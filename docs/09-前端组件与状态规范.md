# 09-前端组件与状态规范

## 1. 文档目标

本文件用于统一前端组件拆分、状态处理和页面实现方式，避免页面风格和交互逻辑快速失控。

---

## 2. 组件分层

前端组件按 4 层拆分：

### 2.1 `ui` 基础组件层

仅负责通用展示，不含业务逻辑。

示例：

- Button
- Input
- Modal
- Drawer
- EmptyState
- Skeleton
- Pagination
- Tag

### 2.2 `shared` 通用业务组件层

可在多个页面复用，带少量业务语义。

示例：

- ToolCard
- TopicSection
- RatingSummary
- ReviewItem
- SearchBar
- FilterPanel

### 2.3 `features` 功能域组件层

按业务域组织。

示例：

- `features/tool`
- `features/topic`
- `features/review`
- `features/admin-tool`

### 2.4 `pages` 页面装配层

只做页面编排，不写复杂业务细节。

---

## 3. 页面状态规范

### 3.1 必备状态

每个核心页面必须处理：

- `loading`
- `empty`
- `error`
- `success`

### 3.2 列表页状态

工具列表页、专题更多页、后台列表页必须支持：

- 初次加载骨架屏
- 无搜索结果空态
- 接口报错重试态
- 翻页切换中状态

### 3.3 表单页状态

登录页、评论提交、后台新增编辑必须支持：

- 提交中按钮禁用
- 表单校验错误
- 接口提交失败
- 成功提示

---

## 4. 数据获取规范

- 页面不直接调用原始 fetch/axios
- 页面只调用 `services/*`
- 页面不直接拼 query string
- 页面不直接处理 token 刷新逻辑

推荐目录：

```text
src/
├── services/
├── features/
├── components/ui/
├── components/shared/
└── app/
```

---

## 5. ToolCard 规范

ToolCard 是全站核心组件，必须统一。

### 5.1 标准字段

- logo
- name
- shortDesc
- tags
- officialUrl
- isFavorited

### 5.2 交互规则

- 点击卡片主体进入详情页
- 点击官网按钮只跳官网，不触发详情跳转
- 点击收藏按钮只处理收藏，不触发卡片跳转

### 5.3 使用范围

- 首页精选
- 工具列表页
- 专题更多页
- 同类推荐区

---

## 6. 搜索与筛选状态规范

### 6.1 URL 同步要求

以下状态必须同步到 URL：

- keyword
- category
- tags
- sort
- page

这样可以保证：

- 可复制链接
- 可刷新页面
- 可回退前进

### 6.2 本地状态与远端状态边界

本地状态负责：

- 输入框草稿值
- 弹窗开关
- 当前选中项

远端状态负责：

- 工具列表数据
- 专题列表数据
- 详情数据
- 评论列表数据

---

## 7. 登录态规范

- `accessToken` 统一保存在同一层存储方案中
- 页面不手写权限判断逻辑，统一由 auth hooks 或 middleware 处理
- 后台页面与前台页面权限逻辑分开

---

## 8. 错误提示规范

### 8.1 用户提示

统一 3 类提示：

- 轻提示：操作成功、收藏成功
- 表单提示：字段级错误
- 页面级提示：接口失败、数据加载失败

针对 `409` 冲突错误，前端必须给出明确业务提示，不允许统一展示“请求失败”。

典型场景：

- 注册时邮箱已被占用
- 收藏时该工具已存在于收藏列表
- 后台创建分类、标签、专题时 slug 或名称冲突

### 8.2 不允许的写法

- 同一类错误有多个文案风格
- 一个页面 toast，一个页面 alert
- 页面直接展示后端原始错误堆栈

---

## 9. 组件命名规范

- 组件名使用 PascalCase
- Hooks 使用 `useXxx`
- Service 使用 `xxxService`
- 页面块组件以 `Section`、`Panel`、`List` 结尾

示例：

- `ToolCard.tsx`
- `TopicSection.tsx`
- `useToolList.ts`
- `toolService.ts`

---

## 10. 视觉一致性约束

前端必须遵守 [07-UI设计规范.md](07-UI设计规范.md)。

重点要求：

- 不新增第二套配色系统
- 不新增第二套按钮样式
- 卡片圆角、阴影、玻璃态参数统一
- 页面动效节奏统一
