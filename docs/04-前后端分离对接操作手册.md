# 04-前后端分离对接操作手册

## 1. 文档目标

本手册用于指导星点评 V1 项目的前后端分离开发、联调和提测流程。

适用范围：

- 前端：Next.js 14 + React 18 + TailwindCSS
- 后端：NestJS 10 + Prisma + MySQL
- 业务范围：首页、工具列表页、专题更多页、工具详情页、个人中心、内部管理后台

本手册解决 3 件事：

1. 前后端分别先做什么
2. 联调时按什么顺序接
3. 出问题时先查哪里

---

## 2. 协作原则

- 前端只依赖接口契约，不直接假设数据库结构。
- 后端只保证接口契约，不直接绑定前端页面细节。
- 所有联调以 Swagger 和本文档定义为准，不以口头描述为准。
- 任何字段变更、接口变更、状态码变更，必须先改文档再改代码。
- 前后端都必须先用 mock 数据自测，再进入真实联调。

---

## 3. 推荐开发顺序

### 3.1 第一阶段：基础能力先打通

前端先完成：

- 路由骨架
- 页面空壳
- 公共布局
- API 请求封装
- 登录态存储方案
- 列表页、详情页、后台页的 mock 展示

后端先完成：

- 项目初始化
- 数据库 schema 与 migration
- Swagger 基础配置
- 认证模块
- 分类、标签、专题只读接口
- 工具列表与工具详情接口

### 3.2 第二阶段：核心业务联调

建议按以下顺序联调：

1. `/api/categories`
2. `/api/tags`
3. `/api/topics`
4. `/api/tools`
5. `/api/topics/:slug/tools`
6. `/api/tools/:id`
7. `/api/tools/:id/recommend`
8. 登录注册接口
9. 收藏接口
10. 评论接口
11. 个人中心接口
12. 后台 CRUD 接口

### 3.3 第三阶段：收口与提测

- 替换全部 mock
- 统一错误提示
- 检查鉴权和路由守卫
- 检查分页、空状态、加载态
- 自测通过后再提测

---

## 4. 前端工作流

### 4.1 页面拆分

前端按页面域拆分，不按接口拆分：

- 首页
- 工具列表页
- 专题更多页
- 工具详情页
- 个人中心
- 管理后台

### 4.2 请求层规范

前端统一封装请求层，例如：

- `lib/request.ts`：统一 fetch/axios 封装
- `services/tool.ts`：工具相关接口
- `services/topic.ts`：专题相关接口
- `services/auth.ts`：认证相关接口
- `services/admin.ts`：后台相关接口

要求：

- 页面组件里不直接拼接 URL
- 页面组件里不直接解析原始响应结构
- 所有接口响应先在 service 层做基础适配

### 4.3 前端联调方式

- 接口未完成时使用 mock 文件
- 接口完成后只替换 service 层，不大改页面结构
- 每接完一个接口，先在页面本地验证成功、空态、错误态 3 种结果

---

## 5. 后端工作流

### 5.1 模块拆分

后端按领域模块拆分：

- auth
- user
- tool
- category
- tag
- topic
- review
- favorite
- admin

### 5.2 接口输出要求

所有接口优先保证：

- 字段名稳定
- 类型稳定
- 状态码稳定
- 分页结构稳定

后端不得在未同步前端的情况下：

- 随意更改字段名
- 将字符串改成数组
- 将对象改成数组
- 修改分页结构
- 改变登录态传递方式

### 5.3 Swagger 要求

每个对前接口必须：

- 有 DTO
- 有参数说明
- 有响应结构说明
- 在 Swagger 可调试

没有写到 Swagger 的接口，不视为可联调接口。

---

## 6. 联调顺序与检查清单

### 6.1 首页联调

依赖接口：

- `GET /api/topics`
- `GET /api/tools?featured=true` 或等价首页精选接口

前端检查：

- 首页精选是否正常展示
- 专题模块是否展示
- `>> 更多AI工具` 是否跳转正确

后端检查：

- `isFeatured` 数据是否可查询
- 专题返回顺序是否稳定
- 未发布工具是否被过滤

### 6.2 工具列表页联调

依赖接口：

- `GET /api/tools`
- `GET /api/categories`
- `GET /api/tags`

前端检查：

- 搜索、分类、标签、排序是否影响查询参数
- 分页切换是否正确
- 空列表是否有空态

后端检查：

- 查询参数是否支持 `keyword/category/tags/sort/page/pageSize`
- 返回分页结构是否统一

### 6.3 专题更多页联调

依赖接口：

- `GET /api/topics/:slug/tools`

前端检查：

- 专题标题和简介是否正确
- 分页、排序、标签筛选是否可用
- 返回的 `topic.name / topic.slug / topic.description` 是否完整并可渲染

后端检查：

- slug 查不到时返回 404
- 只返回该专题已发布工具
- 响应中必须包含 `topic` 对象，而不只是 `list + pagination`

### 6.4 详情页联调

依赖接口：

- `GET /api/tools/:id`
- `GET /api/tools/:id/recommend`
- `GET /api/tools/:toolId/reviews`
- `POST /api/tools/:toolId/reviews`

前端检查：

- 评分总览是否显示
- 评论提交后是否刷新当前评论
- 未登录提交评论是否拦截

后端检查：

- 单用户单工具单评论是否生效
- 评论覆盖更新逻辑是否正确

### 6.5 个人中心联调

依赖接口：

- `GET /api/users/me`
- `GET /api/favorites`
- `GET /api/users/me/reviews`
- `POST/DELETE /api/favorites/:toolId`

前端检查：

- 未登录是否跳登录
- 收藏与评论数据是否同步刷新

### 6.6 后台联调

依赖接口：

- `/api/admin/tools`
- `/api/admin/categories`
- `/api/admin/tags`
- `/api/admin/topics`
- `/api/admin/reviews`

前端检查：

- 列表、创建、编辑、删除、上下架是否闭环
- 表单字段与接口字段是否一致

后端检查：

- admin 权限是否隔离
- draft/published/archived 状态是否正确影响前台可见性

## 7. 登录态与鉴权对接

### 7.1 前台用户

- 登录成功后保存 `accessToken`
- 刷新接口使用 `refreshToken`
- 请求头统一带 `Authorization: Bearer <accessToken>`

### 7.2 后台用户

- 后台接口与前台接口共用认证体系，但必须校验 `role=admin`
- 前端后台页面进入前先校验当前用户角色

### 7.3 401 处理

前端统一流程：

1. 请求返回 401
2. 自动调用 refresh
3. refresh 成功则重试原请求
4. refresh 失败则清空登录态并跳登录页

## 8. 环境与启动方式

### 8.1 前端环境变量

```bash
NEXT_PUBLIC_API_BASE_URL=http://localhost:3000/api
```

### 8.2 后端环境变量

```bash
PORT=3000
DATABASE_URL=mysql://root:password@localhost:3306/aidianping
REDIS_URL=redis://localhost:6379
JWT_ACCESS_SECRET=xxx
JWT_REFRESH_SECRET=xxx
```

### 8.3 本地启动顺序

1. 启动 MySQL
2. 启动 Redis（如启用）
3. 后端执行 migration 和 seed
4. 启动 NestJS
5. 启动 Next.js
6. 打开 Swagger 检查核心接口
7. 打开前端页面进入联调

---

## 9. 问题排查顺序

接口接不上时，按这个顺序查：

1. 路由地址对不对
2. 请求方法对不对
3. 参数名对不对
4. Token 有没有带
5. Swagger 里能不能调通
6. 后端日志有没有报错
7. 数据库有没有对应数据
8. 前端是不是还在读 mock

---

## 10. 提测前最低要求

前端必须满足：

- 所有核心页面已接真实接口
- 加载态、空态、错误态可见
- 不存在写死的 mock 文案

后端必须满足：

- Swagger 可用
- migration 可执行
- 核心接口有基础自测
- 权限与状态位生效

联调完成的定义：

- 首页可浏览
- 列表可筛选
- 更多页可打开
- 详情可评分评论
- 收藏与个人中心可闭环
- 后台可维护工具并影响前台展示
