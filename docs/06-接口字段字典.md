# 06-接口字段字典

## 1. 文档目标

本文件定义星点评 V1 核心接口的请求参数、响应字段、类型、是否必填和业务含义。

使用原则：

- 前端以本文件和 Swagger 为字段基线
- 后端新增或修改字段时必须同步更新本文件
- 文档未声明的字段，不默认进入联调范围

---

## 2. 通用响应结构

### 2.1 标准响应

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | number | 是 | 业务状态码 |
| message | string | 是 | 响应消息 |
| data | object/array/null | 是 | 业务数据 |

### 2.2 分页结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| list | array | 是 | 列表数据 |
| pagination.page | number | 是 | 当前页 |
| pagination.pageSize | number | 是 | 每页数量 |
| pagination.total | number | 是 | 总条数 |
| pagination.totalPages | number | 是 | 总页数 |

## 3. 认证模块

### 3.1 POST `/api/auth/register`

#### Request

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 用户邮箱 |
| password | string | 是 | 登录密码，至少 8 位 |
| nickname | string | 否 | 用户昵称 |

#### Response.data

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user.id | number | 是 | 用户 ID |
| user.email | string | 是 | 用户邮箱 |
| user.nickname | string | 是 | 用户昵称 |
| user.avatar | string | 是 | 头像 URL，空字符串表示未设置 |
| accessToken | string | 是 | 访问令牌 |
| refreshToken | string | 是 | 刷新令牌 |

### 3.2 POST `/api/auth/login`

#### Request

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 用户邮箱 |
| password | string | 是 | 登录密码 |

#### Response.data

与注册接口返回结构一致。

### 3.3 POST `/api/auth/refresh`

#### Request

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| refreshToken | string | 是 | 刷新令牌 |

#### Response.data

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| accessToken | string | 是 | 新的访问令牌 |
| refreshToken | string | 是 | 新的刷新令牌 |

## 4. 首页与工具模块

### 4.1 GET `/api/topics`

#### Query

无

#### Response.data[]

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 专题 ID |
| name | string | 是 | 专题名称 |
| slug | string | 是 | 专题路由标识 |
| description | string | 是 | 专题简介 |
| toolCount | number | 是 | 专题下工具数量 |

### 4.2 GET `/api/tools`

#### Query

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | string | 否 | 搜索关键词 |
| category | string | 否 | 分类 slug |
| tags | string | 否 | 标签 slug，多个逗号分隔 |
| sort | string | 否 | `popularity` / `newest` |
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页数量，默认 20 |
| featured | boolean | 否 | 是否只查首页精选 |

#### Response.data.list[]

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 工具 ID |
| name | string | 是 | 工具名称 |
| logo | string | 是 | Logo URL |
| shortDesc | string | 是 | 卡片简述 |
| tags | string[] | 是 | 标签名称列表 |
| officialUrl | string | 是 | 官网地址 |
| isFavorited | boolean | 是 | 当前用户是否已收藏 |
| category | string | 是 | 主分类 slug |
| popularity | number | 是 | 热度值 |
| publishTime | string | 是 | 发布时间，ISO 字符串 |
| status | string | 否 | 后台场景返回，前台默认不依赖 |
| isFeatured | boolean | 否 | 后台场景返回，前台默认不依赖 |

### 4.3 GET `/api/topics/:slug/tools`

#### Path

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| slug | string | 是 | 专题 slug |

#### Query

与 `/api/tools` 保持一致，但不接收 `featured`。

#### Response.data

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| topic.name | string | 是 | 专题名称 |
| topic.slug | string | 是 | 专题 slug |
| topic.description | string | 是 | 专题描述 |
| list | array | 是 | 工具列表，字段同 `/api/tools` |
| pagination | object | 是 | 分页结构 |

### 4.4 GET `/api/tools/:id`

#### Path

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 工具 ID |

#### Response.data

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 工具 ID |
| name | string | 是 | 工具名称 |
| logo | string | 是 | Logo URL |
| shortDesc | string | 是 | 简要介绍 |
| fullDesc | string | 是 | 完整介绍 |
| primaryUsage | string | 是 | 主要用途 |
| tags | string[] | 是 | 标签列表 |
| officialUrl | string | 是 | 官网地址 |
| category | string | 是 | 主分类 slug |
| reviewCount | number | 是 | 评论数量 |
| averageRating | number | 是 | 平均评分 |
| ratingDistribution | object | 是 | 评分分布对象 |
| isFavorited | boolean | 是 | 当前用户是否已收藏 |
| createdAt | string | 是 | 创建时间 |

### 4.5 GET `/api/tools/:id/recommend`

#### Response.data[]

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 工具 ID |

| updatedAt | string | 是 | 更新时间 |

## 10. 变更规则

- 新增接口：补充完整表格
- 修改字段：修改对应接口表格
- 删除字段：先在文档标注废弃，再删代码
