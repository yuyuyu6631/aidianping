# 10-后端模块职责与表结构说明

## 1. 文档目标

本文件用于明确后端模块边界、核心职责和主要数据表关系，防止模块越界开发。

---

## 2. 模块职责清单

### 2.1 Auth 模块

职责：

- 注册
- 登录
- refresh
- logout
- token 校验

核心表：

- `users`
- `refresh_tokens`

不负责：

- 用户资料编辑
- 收藏业务
- 评论业务

### 2.2 User 模块

职责：

- 获取当前用户信息
- 更新昵称和头像
- 注销账号

核心表：

- `users`

### 2.3 Tool 模块

职责：

- 工具列表查询
- 工具详情查询
- 首页精选查询
- 同类推荐

核心表：

- `tools`
- `tool_categories`
- `tool_tags`
- `tool_topics`

### 2.4 Category 模块

职责：

- 分类查询
- 后台分类 CRUD

核心表：

- `categories`
- `tool_categories`

### 2.5 Tag 模块

职责：

- 标签查询
- 后台标签 CRUD

核心表：

- `tags`
- `tool_tags`

### 2.6 Topic 模块

职责：

- 首页专题列表
- 专题更多页查询
- 后台专题 CRUD

核心表：

- `topics`
- `tool_topics`

### 2.7 Favorite 模块

职责：

- 查询收藏
- 添加收藏
- 取消收藏

核心表：

- `favorites`

### 2.8 Review 模块

职责：

- 查询评论
- 提交评分评论
- 查询我的评论

核心表：

- `reviews`

### 2.9 Admin 模块

职责：

- 工具后台管理
- 分类标签专题管理
- 评论管理

核心表：

- `tools`
- `categories`
- `tags`
- `topics`
- `reviews`

---

## 3. 表结构职责说明

### 3.1 `users`

关键字段：

- `email`
- `password`
- `nickname`
- `avatar`
- `role`

说明：

- `role` 决定是否可访问后台

### 3.2 `tools`

关键字段：

- `name`
- `shortDesc`
- `fullDesc`
- `primaryUsage`
- `officialUrl`
- `popularity`
- `publishTime`
- `status`
- `isFeatured`

说明：

- `status` 控制前台可见性
- `isFeatured` 控制首页精选

### 3.3 `categories`

关键字段：

- `name`
- `slug`
- `description`

### 3.4 `tags`

关键字段：

- `name`
- `slug`

### 3.5 `topics`

关键字段：

- `name`
- `slug`
- `description`
- `sortOrder`

说明：

- `slug` 是专题更多页唯一标识

### 3.6 `favorites`

关键字段：

- `userId`
- `toolId`

约束：

- `@@unique([userId, toolId])`

### 3.7 `reviews`

关键字段：

- `userId`
- `toolId`
- `content`
- `rating`
- `status`

约束：

- `@@unique([userId, toolId])`

说明：

- 同一用户对同一工具仅保留一条评论

### 3.8 `refresh_tokens`

关键字段：

- `userId`
- `tokenHash`
- `expiresAt`
- `revokedAt`

说明：

- 不直接存明文 refresh token

---

## 4. 表关系说明

### 4.1 核心关系

- `users` 1 对多 `favorites`
- `users` 1 对多 `reviews`
- `users` 1 对多 `refresh_tokens`
- `tools` 1 对多 `favorites`
- `tools` 1 对多 `reviews`
- `tools` 多对多 `categories`
- `tools` 多对多 `tags`
- `tools` 多对多 `topics`

### 4.2 关系表

中间表：

- `tool_categories`
- `tool_tags`
- `tool_topics`

要求：

- 中间表不承载复杂业务逻辑
- 只用于映射关系

---

## 5. 查询边界

### 5.1 前台查询边界

前台查询默认只允许返回：

- `status = published` 的工具
- `status = visible` 的评论

### 5.2 后台查询边界

后台查询允许返回：

- 全状态工具
- 全状态评论

---

## 6. 模块调用约束

- `Admin` 可以编排调用其他模块 service
- `Tool` 不直接处理认证逻辑
- `Review` 不直接处理收藏逻辑
- `Auth` 不直接处理评论和收藏逻辑

如果出现跨模块业务，优先抽 service，不要把控制器互相调用。
