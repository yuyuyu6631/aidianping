# 14-版本变更记录与决策记录

## 1. 文档目标

记录项目中的关键范围调整、架构决策和协作约定，避免后续“忘了为什么这么做”。

## 2. 版本变更记录

### 2026-04-16

#### 变更 1：明确“更多页”定义

- 原问题：更多页语义不清，可能被理解成全站工具库页
- 最终决定：更多页是首页专题模块的二级聚合页
- 影响：
  - 新增专题路由
  - 新增 Topic 模块
  - 首页与专题页形成明确跳转关系

#### 变更 2：V1 引入内部管理后台

- 原问题：AI 工具无法持续维护
- 最终决定：V1 增加内部可用的基础 CRUD 后台
- 范围：
  - 工具 CRUD
  - 分类/标签/专题 CRUD
  - 上下架
  - 首页精选配置
  - 轻量评论处理

#### 变更 3：评论升级为评分 + 评论

- 原问题：只有评论，不利于工具质量判断
- 最终决定：详情页支持评分总览和提交评分评论
- 业务规则：
  - 一人一工具一条评论
  - 重复提交视为覆盖更新

#### 变更 4：技术栈收敛

- 原问题：前端技术栈写成 Vue/React 混合候选
- 最终决定：前端固定为 React 18 + Next.js 14 + TailwindCSS

#### 变更 5：Refresh Token 落库

- 原问题：架构文档写了 refresh token 存库，但 schema 没承接
- 最终决定：增加 `refresh_tokens` 表，使用 tokenHash 存储

#### 变更 6：V1 详情页路由与接口主键统一为 id

- 原问题：历史文档中同时出现 `/tools/[slug]` 和 `/tools/[id]`
- 最终决定：当前 V1 页面路由和接口主键统一以 `id` 为准
- 影响：
  - 前台详情页路由使用 `/tools/[id]`
  - API 使用 `/api/tools/:id`
  - 历史 `slug` 方案仅保留在早期规划文档中作为参考，不进入当前联调基线

#### 变更 7：01 / 02 文档降级为历史规划参考

- 原问题：早期 v2 文档与当前 V1 基线发生明显冲突
- 最终决定：`01-v2开发基础基线.md`、`02-协作知识库.md` 保留，但显式标记为历史规划文档
- 当前基线：
  - 执行手册以 `04-*` 为准
  - 约束手册以 `05-*` 为准
  - 接口字段以 `06-*` 为准
  - 页面路由以 `08-*` 为准

## 3. 架构决策记录（ADR 简版）

### ADR-001：前后端分离而不是单体模板混写

决策：

- 前端使用 Next.js
- 后端使用 NestJS
- 通过 REST API 对接

原因：

- 前后端可并行开发
- UI 节奏与后端业务节奏解耦
- 便于后续独立部署和后台扩展

代价：

- 联调成本更高
- 文档要求更严格

### ADR-002：更多页采用专题模型

决策：

- 引入 `Topic` 和 `ToolTopic`

原因：

- 首页专题模块必须有稳定的承接页
- 专题不等于分类，适合运营组织内容

代价：

- 数据结构比纯分类稍复杂

### ADR-003：后台审核仅做轻量状态位

决策：

- V1 只保留 `draft/published/archived`

原因：

- 用户要求内部 CRUD 优先
- 复杂审批流短期没有价值

代价：

- 后续若做外部投稿，需要再次扩展

### ADR-004：评论采用覆盖更新

决策：

- `reviews` 增加 `@@unique([userId, toolId])`

原因：

- 简化前端展示和聚合统计
- 避免同一用户刷多条评论

代价：

- 无法保留用户历史评论版本

## 4. 协作决策记录

### 4.1 文档优先级

冲突时按以下优先级判断：

1. 对接约束手册
2. 系统架构文档
3. PRD
4. 代码现状

### 4.2 联调原则

- 先文档，后代码
- 先 mock，后联调
- 先 Swagger 可调，后页面接入

### 4.3 接口冻结原则

- 提测前冻结接口结构
- 不允许边测边改字段名和分页结构
