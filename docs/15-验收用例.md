# 星点评 V1 验收用例

## 文档目标

本文档用于前后端联调完成后的验收测试，验证所有核心功能是否按文档约定实现。

使用方式：
- 验收人员按用例逐项执行
- 记录实际结果与预期是否一致
- 发现问题立即阻断，不遗漏到下一阶段

---

## 一、接口契约验收

### 1.1 响应结构固定

**用例编号**：API-001

| 检查项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 所有接口返回 `code` 字段 | 必返回 number | 调用任意接口检查 |
| 所有接口返回 `message` 字段 | 必返回 string | 调用任意接口检查 |
| 所有接口返回 `data` 字段 | 必返回 object/array/null | 调用任意接口检查 |
| 不存在 `result` 等其他根字段 | 结构统一 | 全局搜索代码 |

**验证接口**：任选 3 个以上接口抽查

---

### 1.2 分页结构固定

**用例编号**：API-002

| 检查项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 列表接口返回 `list` | array，非 `items` | 调用 `/api/tools` |
| 分页结构包含 `page` | number | 检查响应 |
| 分页结构包含 `pageSize` | number | 检查响应 |
| 分页结构包含 `total` | number | 检查响应 |
| 分页结构包含 `totalPages` | number | 检查响应 |

**验证接口**：`/api/tools`、`/api/favorites`、`/api/tools/:toolId/reviews`

---
