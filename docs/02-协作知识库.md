# aidianping v2 协作知识库

本文档是开发者 A（前端）和开发者 B（后端）的共享知识基线。

---

## 1. 项目概述

**产品目标**：AI 工具目录站，用户可搜索/浏览 AI 工具，查看详情、评分和评论。

**v2 范围**：首页、工具列表页、工具详情页

**技术架构**：Monorepo（npm workspaces）

```
aidianping/
├── apps/web          # 前端（Next.js）
├── apps/api          # 后端（待初始化）
├── packages/contracts # 共享类型与契约
└── docs/             # 文档
```

---

## 2. 技术栈

| 模块 | 技术选型 |
|------|---------|
| 前端框架 | Next.js + TypeScript + Tailwind CSS |

## 3. 数据模型（来自 contracts）

### 工具相关

| 类型 | 用途 |
|------|------|
| `ToolSummary` | 列表页展示的工具摘要 |
| `ToolDetail` | 详情页完整信息 |
| `ToolTag` | 标签（slug + name）|

### 枚举类型

| 类型 | 可选值 |
|------|--------|
| `PricingType` | free / freemium / paid / contact |
| `PlatformType` | web / ios / android / desktop / chrome-extension / api |
| `SortMode` | hot / latest / rating / name |
| `ViewMode` | card / list |

### 筛选维度

- 分类（category / categorySlug）
- 标签（tags）
- 价格类型（pricing）
- 平台（platforms）
- 语言（language）

## 4. API 契约

### 4.1 工具列表 `GET /api/tools`

**请求参数**（Query）：

| 参数 | 类型 | 说明 |
|------|------|------|
| keyword | string? | 搜索关键词 |
| category | string? | 分类 |
| tags | string[]? | 标签 |
| pricing | string[]? | 价格类型 |
| platforms | string[]? | 平台 |
| language | string[]? | 语言 |
| sort | SortMode? | 排序方式 |
| view | ViewMode? | 视图模式 |
| page | number? | 页码 |
| pageSize | number? | 每页数量 |

**响应**：

```typescript
{
  results: ToolSummary[];
  facets: {
    categories: FacetOption[];
    tags: FacetOption[];
    pricing: FacetOption[];
    platforms: FacetOption[];
    languages: FacetOption[];
  };
  pagination: Pagination;
}
```

### 4.2 工具详情 `GET /api/tools/:slug`

**响应**：`ToolDetail`

### 4.3 AI 搜索 `POST /api/tools/search`

**请求**：

```typescript
{
  query: string;
  page?: number;
  pageSize?: number;
}
```

**响应**：

```typescript
{
  intent: AiIntent;        // 理解的用户意图
  aiPanel: AiPanel;       // AI 推荐面板
  results: ToolSummary[];
  facets: ...;
  pagination: Pagination;
}
```

### 4.4 评分与评论

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建评论 | POST | `/api/tools/:slug/reviews` | body: CreateReviewRequest |
| 获取评论列表 | GET | `/api/tools/:slug/reviews` | - |
| 提交评分 | POST | `/api/tools/:slug/rating` | body: CreateRatingRequest |

## 5. 职责边界

### 开发者 A（前端）

- `apps/web` 的全部开发
- 页面路由与渲染
- 搜索状态与 URL 同步
- 组件开发：工具卡片、筛选条、评论区
- 接入 API（等后端 mock 数据就绪）

### 开发者 B（后端）

- `apps/api` 的全部开发
- `packages/contracts` 的维护（如需新增类型）
- mock 数据准备
- API 接口实现
- AI 搜索逻辑

### 共同职责

- `packages/contracts` 的类型定义（协商确定）
- 接口契约定义
- 文档维护

---

## 6. 开发流程

### 阶段划分

```
Day 1: 初始化 + 锁定信息架构
Day 2: 前端骨架（静态页面）
Day 3: 前端骨架 + mock 数据结构
Day 4: 接口接入
Day 5: 收尾 + 边界状态处理
```

### 代码提交流程

1. 各自从 `main` 创建功能分支
2. 完成开发后提 PR
3. 协商评审后合并
4. 保持 `main` 分支可运行

### 沟通机制

- 重大决策需同步到知识库文档
- 接口变更优先通知对方
- 阻塞事项及时提出

---

## 7. 关键约束

1. **字段命名必须来自 contracts**，不允许私自定义接口字段
2. **页面状态优先由 URL 驱动**
3. **每一阶段结束需保留可运行结果**
4. **不直接复用旧项目页面代码**
