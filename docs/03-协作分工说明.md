# 协作分工说明

## 仓库结构

```
d:/workspace/codespace/
├── aidianping/       [main]          # 主仓库，保留分支，用于合并
├── aidianping-fe/    [feat/frontend] # 前端开发目录
└── aidianping-be/    [feat/backend]  # 后端开发目录
```

## Git Worktree 说明

我们使用 Git Worktree 实现前后端并行开发，互不干扰。

### 常用命令

```bash
# 查看所有 worktree
git worktree list

# 创建新的 worktree
git worktree add ../目录名 -b 分支名

# 删除 worktree（不会删除分支）
git worktree remove ../目录名
```

### 分支管理

| 目录 | 分支 | 用途 |
|------|------|------|
| aidianping | main | 主分支，用于合并代码 |
| aidianping-fe | feat/frontend | 前端功能分支 |
| aidianping-be | feat/backend | 后端功能分支 |

## 开发流程

1. **各自在对应目录开发**
   - 前端 → `aidianping-fe/`
   - 后端 → `aidianping-be/`

2. **完成后提 PR**
   - feat/frontend → main
   - feat/backend → main

3. **合并到 main 后**
   - 可选择删除对应的 worktree
   - 或继续用于后续开发

## 目录切换

```bash
# 切换到前端目录
cd /d/workspace/codespace/aidianping-fe

# 切换到后端目录
cd /d/workspace/codespace/aidianping-be

# 切换回主目录
cd /d/workspace/codespace/aidianping
```

## 注意事项

- 同一文件不能同时在两个分支修改，会冲突
- 每次开始新任务前，先 `git pull origin main` 拉取最新代码
- 接口变更需同步通知对方更新
